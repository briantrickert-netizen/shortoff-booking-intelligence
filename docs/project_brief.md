# Project Brief: Shortoff Booking Intelligence Dashboard

## Business Context

Shortoff Mountain Retreats is a privately owned vacation rental business. Guests may discover the property through Google, direct website visits, social media, Airbnb, referrals, or other channels.

The business needs a clearer way to compare marketing activity, website behavior, direct-booking interest, and platform revenue.

## Problem

Booking and marketing data often live in separate systems. Website traffic may be tracked separately from Lodgify/direct bookings and Airbnb bookings. That makes it harder to determine which channels are creating real business value.

The business needs a practical dashboard that connects traffic behavior with booking-channel economics.

## Objective

Build a lightweight booking intelligence dashboard that compares:

- Website traffic sources
- Booking-intent events
- Direct/Lodgify bookings
- Airbnb-style bookings
- Revenue by channel
- Average booking value
- Monthly revenue trends

## Decision Questions

This project helps answer:

1. Which traffic sources create the most booking intent?
2. Which booking platform produces the most revenue?
3. Which platform has stronger average booking value?
4. Are Airbnb review clicks acting as a trust signal?
5. Where should marketing and website-improvement effort be focused?

## MVP Scope

The MVP includes:

- Synthetic sample website event data
- Synthetic Lodgify/direct booking data
- Synthetic Airbnb-style booking data
- Python cleaning scripts
- Jupyter analysis notebook
- SQL metrics
- Chart screenshots
- GitHub documentation

## Excluded From MVP

The MVP does not include:

- Live API integrations
- Real guest data in GitHub
- Automated reporting
- Paid analytics tools
- Machine learning
- Chatbot functionality

## Privacy Approach

The public GitHub version uses synthetic data only. Real guest data, exact booking records, and revenue exports are excluded.

A private business version can use real data locally or in a private repository.
